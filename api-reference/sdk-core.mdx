## bryelSpanProcessor

```ts
bryelSpanProcessor(options): SpanProcessor
```

A batching span processor that ships spans to bryel **without mapping** — for any
app already emitting OpenInference/OTel spans. Add it to your provider's
`spanProcessors`.

## buildSpanProcessor

```ts
buildSpanProcessor(mapper: SpanMapper, options): SpanProcessor
```

Like `bryelSpanProcessor`, but runs a `SpanMapper` first — promote framework
spans into OpenInference before export.

## buildExporter

```ts
buildExporter(options): SpanExporter
```

The raw OTLP/HTTP-protobuf exporter pointed at the bryel ingest, with the API key
on the `Authorization` header. Compose it yourself if you need a custom processor.

## Options

<ParamField path="apiKey" type="string" required>Project API key (`bk_…`).</ParamField>
<ParamField path="endpoint" type="string" default="https://ingest.bryel.ai/v1/traces">Override for self-hosting.</ParamField>
<ParamField path="headers" type="Record<string, string>">Extra headers, merged after auth.</ParamField>

## SpanMapper

<ParamField path="name" type="string" required>Identifier for the mapper.</ParamField>
<ParamField path="shouldMap" type="(span) => boolean" required>Skip spans this mapper doesn't own.</ParamField>
<ParamField path="map" type="(span) => Attributes" required>OpenInference attributes to **merge** onto the span (does not mutate).</ParamField>
<ParamField path="isTraceRoot" type="(span) => boolean">Optional. Marks an operation root whose framework parent is dropped, so runs root cleanly.</ParamField>
