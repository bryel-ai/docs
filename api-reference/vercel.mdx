API reference for the adapter's exports and options. For end-to-end setup, see the [Vercel guide](/frameworks/vercel).

## Install

```bash
npm i @bryel/vercel @vercel/otel
```

## bryelVercelProcessor

```ts
bryelVercelProcessor(options): SpanProcessor
```

Returns a `SpanProcessor` that maps Vercel AI SDK spans into OpenInference and exports them to bryel. Pass it to `registerOTel`'s `spanProcessors`.

```ts instrumentation.ts
import { registerOTel } from "@vercel/otel";
import { bryelVercelProcessor } from "@bryel/vercel";

export function register() {
  registerOTel({
    serviceName: "my-app",
    spanProcessors: [
      bryelVercelProcessor({ apiKey: process.env.BRYEL_KEY! }),
    ],
  });
}
```

### Options

<ParamField path="apiKey" type="string" required>
  Project API key (`bk_…`). Resolves to your project and tenant server-side.
</ParamField>

<ParamField path="endpoint" type="string" default="https://ingest.eu.bryel.ai/v1/traces">
  OTLP/HTTP traces endpoint. Override for self-hosting.
</ParamField>

<ParamField path="headers" type="Record<string, string>">
  Extra headers, merged after the auth header.
</ParamField>

## Telemetry metadata

Enable telemetry per call and attach metadata through `experimental_telemetry`:

```ts
await generateText({
  model: openai("gpt-4o"),
  experimental_telemetry: {
    isEnabled: true,
    metadata: { sessionId, userId },
  },
  prompt,
});
```

<Note>You set `isEnabled: true` yourself per call — the adapter does not inject it.</Note>

The following `metadata` fields are recognized:

<ParamField path="sessionId" type="string">
  Groups calls into a single conversation.
</ParamField>

<ParamField path="userId" type="string">
  Associates the call with an end user.
</ParamField>

<ParamField path="messageId" type="string">
  A `messageId` you mint, used to attach [feedback](/guides/feedback).
</ParamField>
