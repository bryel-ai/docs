---
title: OpenTelemetry
description: "Add the bryel span processor to any OpenTelemetry-instrumented app."
---

bryel is OpenTelemetry-native. If your app already emits [OpenInference](/concepts) spans, point the bryel processor at your tracer provider and you're done. No framework lock-in.

<Steps>
  <Step title="Install">
    Add the SDK alongside your existing OpenTelemetry packages.

    ```bash
    npm i @bryel/sdk-core @opentelemetry/sdk-trace-base
    ```
  </Step>

  <Step title="Register the processor">
    `bryelSpanProcessor` batches and ships spans to bryel with no mapping. Register it on your tracer provider.

    <CodeGroup>
    ```ts @vercel/otel
    import { registerOTel } from "@vercel/otel";
    import { bryelSpanProcessor } from "@bryel/sdk-core";

    registerOTel({
      serviceName: "my-app",
      spanProcessors: [bryelSpanProcessor({ apiKey: process.env.BRYEL_KEY })],
    });
    ```

    ```ts NodeSDK
    import { NodeSDK } from "@opentelemetry/sdk-node";
    import { bryelSpanProcessor } from "@bryel/sdk-core";

    new NodeSDK({
      spanProcessors: [bryelSpanProcessor({ apiKey: process.env.BRYEL_KEY })],
    }).start();
    ```
    </CodeGroup>
  </Step>

  <Step title="Set your key">
    Spans batch and POST to bryel ingest with `Authorization: Bearer <key>`. Set `BRYEL_KEY` in your environment.

    ```bash
    BRYEL_KEY=...
    ```
  </Step>
</Steps>

## Custom mapping

If your spans aren't OpenInference yet, supply a `SpanMapper`. This is exactly what `@bryel/vercel` does for the AI SDK.

```ts
import { buildSpanProcessor, type SpanMapper } from "@bryel/sdk-core";

const myMapper: SpanMapper = {
  name: "my-framework",
  shouldMap: (span) => span.name.startsWith("my."),
  map: (span) => ({ /* OpenInference attributes to merge */ }),
};

const processor = buildSpanProcessor(myMapper, { apiKey: process.env.BRYEL_KEY });
```

<Tip>
  See [`@bryel/semconv`](/api-reference/semconv) for the convention constants.
</Tip>
