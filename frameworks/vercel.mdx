---
title: Vercel AI SDK
description: "Trace the Vercel AI SDK with one span processor and a per-call flag."
---

`@bryel/vercel` maps the Vercel AI SDK's OpenTelemetry spans into OpenInference and ships them to bryel — models, tokens, cost, tool calls, and the full multi-step trajectory. You never wrap your model calls.

<Steps>
  <Step title="Install">
    Add the bryel processor and the Vercel OTel package.

    ```bash
    npm i @bryel/vercel @vercel/otel
    ```
  </Step>

  <Step title="Register the processor once">
    Add `bryelVercelProcessor` to `registerOTel` in `instrumentation.ts`. Next.js calls this on startup.

    ```ts instrumentation.ts
    import { registerOTel } from "@vercel/otel";
    import { bryelVercelProcessor } from "@bryel/vercel";

    export function register() {
      registerOTel({
        serviceName: "my-app",
        spanProcessors: [bryelVercelProcessor({ apiKey: process.env.BRYEL_KEY! })],
      });
    }
    ```
  </Step>

  <Step title="Enable telemetry per call">
    Set `experimental_telemetry.isEnabled` on each model call you want traced.

    ```ts
    const result = streamText({
      model: openai("gpt-5"),
      prompt,
      stopWhen: stepCountIs(10),
      tools: { /* … */ },
      experimental_telemetry: {
        isEnabled: true,
        metadata: { sessionId, userId },
      },
    });
    ```
  </Step>

  <Step title="Verify">
    Run a call, then open the trace explorer. You see the model, tokens, cost, and tool calls for the run.
  </Step>
</Steps>

## Grouping and feedback

The `metadata` you pass becomes how bryel groups and joins data.

<ParamField path="sessionId" type="string">
  Groups multiple turns into one conversation.
</ParamField>

<ParamField path="userId" type="string">
  The end-user behind the run.
</ParamField>

<ParamField path="messageId" type="string">
  A per-turn id **you mint** (`crypto.randomUUID()`). bryel stamps it onto the
  spans as `bryel.interaction.id` so [feedback](/guides/feedback) joins back to
  the right turn. Return it to your client to attach 👍/👎.
</ParamField>

```ts
const messageId = crypto.randomUUID();
const result = streamText({
  model: openai("gpt-5"),
  experimental_telemetry: { isEnabled: true, metadata: { sessionId, userId, messageId } },
});
return result.toTextStreamResponse({ headers: { "x-bryel-message-id": messageId } });
```

<Warning>A streamed span finalizes only once the stream is fully consumed — always drain it.</Warning>

For full processor options, see the [`@bryel/vercel` reference](/api-reference/vercel).
